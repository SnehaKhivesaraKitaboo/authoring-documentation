Table of Contents (TOC) Component

The TOC is the central navigation panel in KITABOO Authoring that manages the full lesson structure — Instructions pages, Steps, Level-of-Support (LOS) variants, and chapter headings. It drives page creation, ordering, deletion, and content persistence across the authoring session.

Overview

The Table of Contents (TOC) is the left-hand navigation panel in KITABOO Authoring. It is activated when TOCFlag === 'flat' and driven entirely by the AngularJS newToc directive defined in editor/ngdragdrop.js.

The TOC renders a hierarchical view of the lesson content:

  • Instructions for Students — fixed first entry.
  • Instructions for Teacher — fixed second entry.
  • Steps — dynamically grouped by metadata.stepid; each step can have multiple Level-of-Support (LOS) page variants: CORE, LIGHT, MODERATE, INTENSIVE.
  • Chapter Headings — canvas-side headings persisted back to TOC via saveTOCtoBackend().

Key Capabilities:

  • Click any TOC row to navigate to that page (auto-saves current page first)
  • Add new steps via "+ Add" dropdown
  • Import steps from the Asset Library
  • Drag-and-drop reordering of steps
  • Clone a step into a new LOS level
  • Edit step metadata (title, type, LOS)
  • Delete a page (CORE only if no other LOS variants exist)
  • Move pages Up / Down within a step

Data Model

Flat Page Object ($rootScope.tocInfo[])

Each entry returned by the backend and stored in tocInfo:

{
  id: <pageDbId>,
  pageSequence: <number>,
  originalPageSequence: <number>,
  pageReferenceId: <string>,
  pageType: "studentInstructions" | "teacherInstructions" | <step-page>,
  metadata: {
    identifier: "fill-in-the-blank" | "multiple-choice-template" | ...,
    name: "Fill in the Blanks" | ...,
    stepTitle: "Step title text",
    stepid: <number>,          // Groups pages into Steps
    levelsOfSupport: ["CORE" | "LIGHT" | "MODERATE" | "INTENSIVE"],
    pageTrayIcon: <optional override SVG path>,
    studentInstructions: "<text>",
    supportContentJson: [...],
    clicData: [...]
  },
  headers: [...]              // Chapter heading objects on this page
}

Grouped TOC ($rootScope.grouptoc)

The UI-facing model built by createGroupTOC():

$rootScope.grouptoc = {
  studentInstructions: { id, pageSequence, originalPageSequence, ... },
  teacherInstructions: { id, pageSequence, originalPageSequence, ... },
  steps: [
    // Step 1 — array of LOS variants
    [ pageObj_CORE, pageObj_LIGHT, pageObj_MODERATE ],
    // Step 2
    [ pageObj_CORE ],
    // Step N...
  ]
}

The template iterates: ng-repeat="step in grouptoc.steps" for the outer step rows and ng-repeat="pages in step" for the LOS page rows within each step.

Delete Guard: canDeletePage(pages)

Returns false (delete hidden) when the page is the CORE variant and other LOS variants exist within the same step. CORE is the canonical page — it cannot be removed while dependent LOS pages (LIGHT, MODERATE, INTENSIVE) remain. Delete is allowed only for:

  • Non-CORE LOS pages at any time
  • CORE page only when it is the sole page in that step

Known Behaviors & Design Notes

  • Auto-save on navigate: Clicking any TOC row triggers savePageContent() and saveTOCtoBackend() before the new page loads — no explicit "Save" button needed.
  • Heading sync: Canvas headings (.chapter-headingNew-h1, [data-type="header"]) are scanned and synced to TOC on every page save, keeping the TOC chapter list in sync with canvas content.
  • CORE-first rule: The CORE LOS variant must always be the first page created in a step. Other LOS variants (LIGHT, MODERATE) are derived from or cloned after CORE.
  • Drag-sort scope: Only the Steps list (.toc-steps-list) is drag-sortable. The Instructions for Students and Instructions for Teacher entries are fixed positions.
  • Internal hyperlinks: createRefidmap() populates window.orgSequenceRefidmap used by the internal hyperlink picker to display page titles by reference ID.
  • Full refresh after mutation: Every add, delete, or clone operation calls getTOCData() again to reload the authoritative server state — there is no optimistic local update.
  • TOCFlag check: The entire TOC panel is wrapped in ng-if="TOCFlag==='flat'" — books configured with a different TOC mode will not render this panel.
