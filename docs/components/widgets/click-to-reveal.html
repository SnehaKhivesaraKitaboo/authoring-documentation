Click to Reveal Component

Hide and reveal content sections with interactive click actions.

Overview UML Class Diagram Sequence Diagram Component Flow Features

Overview

The Click to Reveal component provides an interactive way to progressively disclose information. Users click triggers or buttons to reveal hidden content, making it ideal for step-by-step instructions, spoilers, and gradual information disclosure.

Interaction Flow:

  • Authoring Phase: Author configures widget settings, uploads content, and saves configuration to backend
  • Student Phase: Student loads widget, interacts with content, and submits answers for validation
  • Validation Phase: System validates student input, provides feedback, and manages attempt tracking
  • Persistence: State is continuously saved to SCORM API for progress tracking and session recovery

Flow Summary:

  • Editor Flow: Authors create reveal sections (1-10) with trigger text and hidden content, optionally add media (images, videos, audio), and select from 4 visual styles with layout customization.
  • Student Flow: Students click trigger buttons to expand sections and reveal hidden content, with accordion or multi-expand modes supporting flexible navigation patterns.
  • Validation: Non-assessed content delivery focuses on exploration tracking, marking sections as viewed and checking completion status when all sections have been accessed.
  • Data Flow: Tracks viewed sections array and completion progress, persisting view state to SCORM with media initialization for embedded audio/video players.
