Click to Reveal Component

Progressive disclosure widget with a button that reveals hidden content on click. Supports 4 visual styles, optional media, and configurable color theming.

OverviewUML ClassSequenceComponent FlowFeaturesTechnicalData ModelFilesSettingsBehaviors

Overview

The Click to Reveal Component presents a configurable reveal button that students click to expose hidden content. Authors configure button text, media placement, color theming across 7 color pickers, and one of 4 visual styles. The reveal is one-way — content cannot be re-hidden once shown. No SCORM state is tracked; the revealed state resets on page reload.

Key Capabilities

  • 4 distinct visual styles (CTR_Style1 through CTR_Style4)
  • 7 color pickers: card background, header, line, outline, button background, outline, text
  • Optional media: image, video, or audio with 4 placement options
  • Button text configurable up to 50 characters
  • Left/Right placement only available for images (not video/audio)
  • Video closed caption and transcript support with VTT upload

Behaviors and Constraints

  • One-way reveal: Once introClick() fires, the button is hidden and .ClickToRevealData is shown. There is no mechanism to re-hide the content. Page refresh returns to initial state.
  • No SCORM: The component does not track revealed state via SCORM. Each page load starts with the button visible regardless of prior interaction.
  • Left/Right placement restriction: Left and rightSide placement options are only functional for image media. Selecting them for video or audio has no effect.
  • Style color adjustment: ApplystyleCTR() recalculates and applies color variables when the style variant changes, potentially overriding custom color picker values.
  • Video tracks token auth: Video closed-caption track files use token-based authentication URLs; tokens may expire for offline/packaged content.
