Annotation Component

Interactive annotation system that allows users to add comments, notes, and highlights to content. Perfect for collaborative learning and document review.

Overview UML Class Diagram Sequence Diagram Component Flow Features Technical Details

Overview

The Annotation Component provides a rich, interactive interface for adding contextual notes, comments, and highlights to educational content. It supports multiple annotation types including text notes, inline comments, and highlighted sections.

Key Capabilities:

  • Add inline text annotations
  • Create comment threads
  • Highlight important sections
  • Collaborative annotation support
  • Export annotations as summary

Sequence Diagram

This diagram shows the interaction sequence between different actors and system components:

sequenceDiagram participant Author participant W as Annotation Widget participant UI as UI Layer participant API as Backend API participant Student participant SCORM as SCORM API Note over Author,API: Authoring Phase Author->>W: Open Editor W->>UI: Render Settings Panel Author->>W: Enable Highlights/Outlines Author->>W: Select Colors (up to 5 each) Author->>W: Add Custom Labels Author->>W: Set Alt Text W->>W: Validate Configuration W->>API: Update Annotation Metadata API->>API: Save colorObj to activityMetadata API-->>W: Configuration Saved Note over Student,SCORM: Student Interaction Phase Student->>W: Load Widget in Reader W->>UI: Render Annotation Options UI->>Student: Display Color Palette Student->>W: Select Text W->>UI: Show Annotation Menu Student->>W: Choose Highlight/Outline Color W->>UI: Apply Annotation Style W->>SCORM: Save Annotation State SCORM-->>W: State Persisted Student->>W: Continue Annotating loop Multiple Annotations Student->>W: Select and Annotate Text W->>SCORM: Update State end

Interaction Flow:

  • Authoring Phase: Author configures widget settings, uploads content, and saves configuration to backend
  • Student Phase: Student loads widget, interacts with content, and submits answers for validation
  • Validation Phase: System validates student input, provides feedback, and manages attempt tracking
  • Persistence: State is continuously saved to SCORM API for progress tracking and session recovery
