Annotation Component

Interactive text annotation tool for configuring highlight and outline color sets. Authors define color palettes; students annotate reading passages with color-coded highlights and outlines.

Overview UML Class Sequence Component Flow Features Technical Data Model Files Settings Behaviors

Overview

The Annotation Component enables authors to configure two independent color palettes — one for text highlights and one for text outlines. Students use these palettes to annotate reading passages, marking important words or phrases with up to 5 highlight colors and up to 5 outline colors. The component integrates directly with page-level metadata rather than component-level JSON, making annotation data part of the page's activity metadata.

Key Capabilities

  • Up to 5 highlight colors and 5 outline colors, each with a custom label
  • Independent enable/disable toggles for highlights and outlines
  • Alt text support for accessibility (max 2000 characters)
  • Debounced auto-save (1 second) plus explicit save on navigation
  • Metadata stored in page activityMetadata.colorObj (not component JSON)
  • Live color preview rendered via createPreviewForAnnotation()
  • Pastel palette for highlights; dark palette for outlines

Behaviors & Constraints

  • Minimum 1 type enforced: checkHighlightsToggle() prevents both showHighlights and showOutline from being false simultaneously.
  • Unique colors per type: Each color within highlights or outlines must be unique; duplicate prevention enforced in selectHighlightsColor().
  • Page-level storage: Color data is stored in activityMetadata.colorObj on the page, NOT in the component JSON file — missing in component exports.
  • $$hashKey stripped: AngularJS $$hashKey properties are removed from the payload before the PUT API call to prevent server-side issues.
  • Debounced save: saveAnnotationInit() uses a 1-second debounce; additionally triggers on page navigation events.
  • Pastel vs dark palettes: Highlight colors use a pastel color palette; outline colors use a dark/saturated palette for visual contrast.
  • updateAnnotationMetadata(): Defined at NGController line 7065; called by the directive to push colorObj changes to the page metadata store.
