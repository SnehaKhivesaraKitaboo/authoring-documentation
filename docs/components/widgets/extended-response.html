Extended Response (ITC)

Extended Response Component

Long-form text response component for essay-style answers.

Overview Architecture Use Cases Features

Overview

The Extended Response component provides a rich text editor for learners to compose lengthy, essay-style answers. It supports multimedia integration, formatting tools, word count tracking, and can be used for open-ended questions requiring detailed explanations.

User Workflows:

  • Learner: Reads the prompt, composes detailed response in the rich text editor, applies formatting and adds media if needed, reviews word count, and submits the response.
  • Author: Creates essay prompts or open-ended questions, defines evaluation rubrics with criteria, sets constraints (word limits, time limits), and configures auto-save settings.

Flow Summary:

  • Editor Flow: Authors add media content (image, video, or audio), set question text, choose input type (short answer or RTE), configure character limits, and optionally select custom language keyboards.
  • Student Flow: Students view media content, read questions, type responses in text fields or rich text editors, with character counters tracking input length and enabling submission when complete.
  • Validation: No automated scoring; responses are captured and submitted for manual teacher grading, with validation limited to character count enforcement and content sanitization.
  • Data Flow: Student responses are encoded and stored in SCORM suspend_data, with response time latency tracked for analytics and submission confirmation displayed upon completion.
