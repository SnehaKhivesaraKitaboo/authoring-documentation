Extended Response (ITC) Component

Media-anchored open-ended response widget (Identify the Clip). Students view an image, video, or audio clip and respond via short answer, long answer, or rich text editor with optional exact-match validation.

Overview

The Extended Response (Identify the Clip) Component anchors an open-ended question to a media clip (image, video, or audio). Students respond via short answer, long answer, or a rich text editor. Authors can optionally enable exact-match validation with pipe-separated accepted answers. Math support is available via MathLive. The largest widget script files in the codebase (~2000-2200 lines each).

Key Capabilities

  • 3 answer types: Short (500 chars), Long (5000 chars), RTE (rich text)
  • 3 validation modes: no validation, accept any, exact match (pipe-separated)
  • Media types: image, video (system/YouTube/Vimeo), audio
  • Custom keyboard support for Arabic and other languages
  • Math equation input via MathLive web component
  • Audio playinBackground continues during page navigation
  • 2 visual style variants

Behaviors and Constraints

  • 3 validation modes: No validation (always marks complete), Accept any response (validates non-empty), Exact match (pipe-separated answers normalized by normalizeQuotes()).
  • Pipe-separated answers: Multiple accepted answers separated by pipe: Paris|paris|PARIS. All are compared after normalizeQuotes() standardization.
  • Cazary RTE: The basic RTE is Cazary, an iframe-based editor using designMode=on. Jodit is the full WYSIWYG alternative with 30+ toolbar buttons for ELA.
  • MathLive equations: Math equations use the MathLive web component; equations converted to LaTeX for storage via Temml with LaTeX fallback.
  • playinBackground: When enabled, audio tracks continue playing even when the student navigates to another page in the book.
  • Custom keyboard: inserts characters via execCommand("insertText") for non-Latin keyboards; may not work in all browsers due to execCommand deprecation.
