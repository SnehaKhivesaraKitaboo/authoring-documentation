Image Labeling Component

Drag-and-drop labeling activity where students match numbered or text labels to hotspot positions on an image using configurable arrows and connectors. Supports partial credit and up to 20 labels.

OverviewUMLSequenceFlowFeaturesTechnicalData ModelFilesSettingsBehaviors

Overview

The Image Labeling Component displays a background image with numbered or text hotspot targets. Students drag label chips from a bank to match the correct hotspot. Connector arrows are drawn using calculated angles (Math.atan2) between the label position and its connector anchor point. Supports up to 20 labels with partial credit on Try Again.

Key Capabilities

  • Number mode (1,2,3...) or Text mode (custom labels)
  • Up to 20 labels with percentage-based positions
  • 3 arrow types: Arrow, Circle Arrow, Line Arrow
  • 2 hotspot shapes: Circle or Square
  • Partial credit: correct labels lock on Try Again
  • Arrow angle calculated via Math.atan2(dy, dx)
  • Mobile popup warning shown (activity remains functional)

Behaviors and Constraints

  • Arrow calculation: Connector arrow length is the hypotenuse between box1 and box2 (distance formula); angle is Math.atan2(dy, dx)*(180/Math.PI) converted to CSS rotation degrees.
  • Percentage-based coordinates: All label positions (labelLeft, labelTop, box1left, etc.) are stored as percentage strings for responsive scaling with the image container.
  • Partial credit on TryAgain: Correctly placed labels are locked and cannot be moved; incorrect labels return to the label bank for re-attempt.
  • Mobile popup warning: A popup warning message appears on mobile devices indicating the activity may be difficult to use on small screens, but the activity remains interactive.
  • Number vs Text UI difference: Number mode renders circular numbered chips; Text mode renders rectangular chips with custom labelText and configurable width/height.
