Component Overview

The Word Search component provides an interactive puzzle game where users find hidden words in a letter grid across multiple directions with highlighting and validation.

Interaction Flow:

  • Authoring Phase: Author configures widget settings, uploads content, and saves configuration to backend
  • Student Phase: Student loads widget, interacts with content, and submits answers for validation
  • Validation Phase: System validates student input, provides feedback, and manages attempt tracking
  • Persistence: State is continuously saved to SCORM API for progress tracking and session recovery

Flow Summary:

  • Editor Flow: Authors add words to list (max 13 words, max 13 characters each, alphabetic only), generate puzzle grid with words placed horizontally/vertically/diagonally with random letter fill, optionally shuffle grid for variation, configure settings to show/hide word list, and set max attempts.
  • Student Flow: Students interact with puzzle grid using mouse/touch events (mouse down, move, up) to select letter sequences, with CSS classes for selection feedback, finding words that match the list with adjacent letter validation and consistent orientation checking.
  • Validation: Selected word sequences are validated against the word list, with correct words marked ('partOfCorrect' class) and crossed out, incorrect selections temporarily marked ('partOfIncorrect' class) then cleared, submitting when all words are found with final correctData/incorrectData styling.
  • Data Flow: Correct answer list array tracks found words with completion progress, synchronized with SCORM, with Try Again recreating the same puzzle grid and clearing progress, or Show Me auto-solving (game.solve) and marking all word letters with 'solved' class.
