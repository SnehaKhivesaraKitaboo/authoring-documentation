Word Search Component

Interactive word-search puzzle with a hardcoded 13x13 grid supporting up to 13 words. Features click-and-drag selection, 8-direction placement, and Show Me auto-solve functionality.

Overview UML Sequence Flow Features Technical Data Model Files Settings Behaviors

Overview

The Word Search Component generates an interactive puzzle on a hardcoded 13x13 letter grid. Authors add up to 13 words (max 13 characters each, uppercase) which are placed in the grid by the wordfind.js backtracking algorithm. Students click and drag to highlight words in the grid. A Show Me function reveals all answers and disables further interaction.

Key Capabilities

  • Hardcoded 13x13 grid � fixed size, cannot be changed
  • Up to 13 words, each max 13 uppercase characters
  • 3 of 8 orientations used: horizontal, vertical, diagonal
  • Click-and-drag selection with orientation lock after 2 letters
  • Show Me auto-solves and disables all interaction
  • Grid Shuffle button regenerates letter placement
  • Fill letters exclude q, x, z for readability

Behaviors and Constraints

  • Hardcoded 13x13 grid: The puzzle grid is fixed at 13 rows x 13 columns. This cannot be configured by authors. Words longer than 13 characters cannot be placed and are silently truncated or rejected by wordfind.js.
  • 3 of 8 orientations: The wordfind.js library supports 8 orientations but only horizontal, vertical, and diagonal are used. Reverse and other orientations are disabled in the configuration passed to newPuzzle().
  • Orientation lock after 2 letters: Once a student drags through 2 grid cells, the selection direction is locked to the detected orientation. Changing direction after 2 cells is not possible without releasing and starting again.
  • Fill letter exclusions: The letter fill algorithm excludes q, x, and z from random fill positions to avoid confusing students with rare letters appearing frequently in background positions.
  • Show Me disables interaction: onWsShowMeBtnClick() calls game.solve() which programmatically selects all words and applies the .solved CSS class, then disables all mouse/touch event handlers to prevent further interaction.
  • Undo/redo desync bug: Using browser undo/redo within the word input fields can cause the JavaScript wordList array to fall out of sync with the displayed word inputs and the generated puzzle grid. A save/reload cycle is needed to resync.
  • Incorrect feedback temporary: When a student selects a non-word sequence, cells get the .partOfIncorrect class showing red highlighting briefly, then the class is removed after a short delay.
