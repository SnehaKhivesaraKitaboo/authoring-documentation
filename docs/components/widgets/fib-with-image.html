Image-based fill-in-the-blank questions with visual context.

Overview Architecture Use Cases Features

Overview

The FIB with Image component combines visual content with fill-in-the-blank exercises. Users complete sentences or labels related to an image, making it perfect for anatomy, geography, diagrams, and visual vocabulary activities.

User Workflows:

  • Learner: Views an image with blank labels or captions, identifies the correct terms, fills in the blanks, and receives feedback with correct answers highlighted.
  • Author: Uploads or selects images, positions blank fields on or around the image, defines correct answers for each blank.

Flow Summary:

  • Editor Flow: Authors upload a base image, add positioned answer cards (2-20) with text or image content, insert blanks with dropdown or text input types, and select from 7 card styles.
  • Student Flow: Students view cards overlaid on the base image, fill blanks by selecting dropdown options or typing text answers, with all card blanks required for submission.
  • Validation: Card answers are validated individually with visual feedback (green borders/checkmarks for correct, red borders/crosses for incorrect) and aggregate scoring calculated.
  • Data Flow: Card answer array is maintained and synchronized with SCORM, with Try Again clearing incorrect responses or Show Me revealing all correct answers on the image layout.
