Correction Component

Mark and correct errors in text with inline editing capabilities.

Overview UML Class Diagram Sequence Diagram Component Flow Features

Overview

The Correction component allows learners to identify and correct errors in text passages. It supports grammar, spelling, punctuation corrections, and can provide inline editing or selection-based correction modes.

Interaction Flow:

  • Authoring Phase: Author configures widget settings, uploads content, and saves configuration to backend
  • Student Phase: Student loads widget, interacts with content, and submits answers for validation
  • Validation Phase: System validates student input, provides feedback, and manages attempt tracking
  • Persistence: State is continuously saved to SCORM API for progress tracking and session recovery

Flow Summary:

  • Editor Flow: Authors create sentences (1-5) with errors, mark incorrect words, provide correct word options, and configure attempts (1-5) with custom feedback messages.
  • Student Flow: Students identify and click wrong words to display correction options, select the correct replacement, and see strikethrough formatting applied to errors with corrections shown.
  • Validation: Upon submission, all corrections are validated with visual feedback (green borders and checkmarks for correct, red borders and crosses for incorrect) and scoring calculated.
  • Data Flow: Corrections are stored in a state object, synchronized with SCORM, with Try Again resetting only incorrect answers or Show Me revealing all correct solutions.
