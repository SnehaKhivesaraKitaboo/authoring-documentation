Binary-choice assessment widget sharing the MCQ codebase. Features the same 60 visual styles (6 layouts x 10 colors) and media support, limited to 2 answer choices with single-select validation.

Overview

The True and False Component is a specialized binary-choice assessment that shares most of its codebase with the MCQ component. It presents exactly 2 answer choices (True/False or custom labels) in single-select mode. Both choices share the same 60 visual style combinations as MCQ. The data-type attribute distinguishes True/False from MCQ at runtime. A known bug allows addOptionClick() to add up to 5 options despite the intended limit of 2.

Key Capabilities

  • Exactly 2 answer choices (True and False, or custom text)
  • Always single-select (radio button) mode
  • 60 visual styles: 6 layouts x 10 color presets
  • Individual feedback per option (Base64-encoded)
  • Media support: image, video (local/YouTube), audio
  • Known bug: addOptionClick allows up to 5 choices

Behaviors and Constraints

  • Shared MCQ codebase: The True/False directive reuses most MCQ functions and data structures. Variable names use mcq/mcssData naming. The data-type attribute distinguishes the two at runtime.
  • addOptionClick bug: The addOptionClick() function allows adding up to 5 choice options instead of enforcing the intended 2-choice limit. Authors should not add more than 2 choices.
  • Always single-select: Unlike MCQ which supports multiple-select, True/False always has isSingleSelect=true and renders radio button inputs.
  • Base64 feedback: Per-option feedback is stored as btoa-encoded strings (feedbackinline) and decoded via atob() for display — same mechanism as MCQ.
  • Audio 300ms timeout: When audio media is attached, the Audio.js player initialization uses a 300ms $timeout to ensure the DOM has rendered before the player attaches.
  • 60 visual styles: Identical to MCQ — 6 layout styles x 10 color presets = 60 combinations. Custom color pickers can override any preset.
