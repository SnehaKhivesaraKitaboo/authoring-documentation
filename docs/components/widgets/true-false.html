True and False Component

Binary true/false question format for quick assessments.

Overview Architecture Use Cases Features

Overview

The True and False component provides a simple binary choice assessment format where learners evaluate statements and select whether they are true or false. It's perfect for quick knowledge checks, fact verification, and binary decision-making exercises.

User Workflows:

  • Learner: Reads a statement, evaluates its validity, selects True or False, submits the answer, and reviews feedback with explanations for correct answers.
  • Author: Creates true/false statements, sets the correct answer, provides explanations for why statements are true or false, and configures feedback display options.

Flow Summary:

  • Editor Flow: Authors set question text with optional media (image, video, audio), configure two options (default True/False but editable), mark the correct answer, add individual feedback per option, select visual style (card styles 1-3 or full-bleed styles 1-3), choose from 10 color schemes, select label type (A-B, 1-2, none), and configure attempts with generic feedback.
  • Student Flow: Students view questions with optional media, select one of two radio button options (True/False), with CSS styling applied to selection and submit button enabled upon choice.
  • Validation: Binary validation compares selected vs correct option with visual feedback (green border/checkmark icon for correct, red border/cross icon for incorrect), supporting both individual option-specific feedback and generic overall feedback messages.
  • Data Flow: Selected option index is tracked and compared with correct option index, synchronized with SCORM, with Try Again clearing selection and resetting visuals, or Show Me highlighting the correct answer and disabling interaction.
