Sorting Component

Drag-and-drop sorting activity where students arrange words, phrases, or images into correct order within sentences. Supports 1-5 sentences with embedded drag items and 3 visual styles.

OverviewUMLSequenceFlowFeaturesTechnicalData ModelFilesSettingsBehaviors

Overview

The Sorting Component presents sentences with word or image drag items that students must arrange into the correct order. Drag items are embedded within the sentence HTML using a special data-id format (DI_{rowNo}_{blankNo}). Preview clones items into drop zones using Fisher-Yates shuffle. Supports 1-5 sentences with up to 5 drag items each (25 total maximum).

Key Capabilities

  • 1-5 sentences with embedded drag items (max 25 total)
  • Text or image drag items (configurable per activity)
  • 3 visual style variants (sort-style1, sort-style2, sort-style3)
  • Fisher-Yates shuffle on each load
  • On non-final TryAgain: incorrect items removed, correct stay locked
  • Drop zones implemented via jQuery UI Droppable .sort-with-row
  • eval() used for button handlers (security risk)

Behaviors and Constraints

  • Drag item ID format: Each drag item is embedded in the sentence HTML using the format data-id="DI_{rowNo}_{blankNo}" — e.g., DI_1_1 for the first blank in the first sentence. This ID drives matching on submit.
  • Fisher-Yates shuffle per load: The preview renderer clones drag items into .preview-sort containers using a Fisher-Yates shuffle. Shuffle is applied on every page load — students see different orders each time.
  • TryAgain partial credit: On non-final attempts, correctly placed items remain locked (cannot be moved); incorrect items are returned to the bank for re-attempt.
  • eval() security risk: Button click handlers in the sorting template use eval() to execute dynamic function strings — a known security anti-pattern that could allow code injection if input is not sanitized.
  • dragradioChange: Switching between Text and Image modes calls dragradioChange() which rebuilds the entire drag item list, clearing any unsaved configuration.
