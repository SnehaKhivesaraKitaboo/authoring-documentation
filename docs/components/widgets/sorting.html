Sorting Component

Drag-and-drop sorting component for ordering items sequentially.

Overview UML Class Diagram Sequence Diagram Component Flow Features

Overview

The Sorting component enables users to arrange items in a specific order through an intuitive drag-and-drop interface. It validates the sequence and provides immediate feedback on correct positioning.

Interaction Flow:

  • Authoring Phase: Author configures widget settings, uploads content, and saves configuration to backend
  • Student Phase: Student loads widget, interacts with content, and submits answers for validation
  • Validation Phase: System validates student input, provides feedback, and manages attempt tracking
  • Persistence: State is continuously saved to SCORM API for progress tracking and session recovery

Flow Summary:

  • Editor Flow: Authors create sentences/statements (1-5) with drag items, choose item types (text up to 150 chars or images with alt text), set correct sequential order, and select from 3 visual styles.
  • Student Flow: Students drag randomized items using jQuery UI into drop zones in correct sequence order, with helper clones for drag feedback and validation preventing invalid placements.
  • Validation: Sequence order is validated for each sentence with visual feedback (green borders/checkmarks for correct sequences, red borders/crosses for incorrect), calculating aggregate scoring across all sentences.
  • Data Flow: Item sequence array tracks placement order with item identifiers, synchronized with SCORM, with Try Again returning incorrect items to the pool with reshuffling, or Show Me displaying all correct sequences.
