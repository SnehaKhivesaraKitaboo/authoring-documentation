Flow Summary:

  • Editor Flow: Authors create tables (1-20 rows/columns), edit cells with right-click context menus for row/column operations and cell merging, paste Excel data with MS Office markup sanitization, enable specific cells for student editing with labels, resize columns (min 50px), and select from 4 style themes with advanced Jodit formatting options.
  • Student Flow: Students view tables in read-only mode or click editable cells to open modal editors, type responses with character counting (max 2000), using plain text or Jodit rich text editors based on configuration, and submit when all required cells are filled.
  • Validation: No automated answer validation; student responses are captured for manual grading, with character limit enforcement and HTML sanitization for rich text content ensuring data integrity.
  • Data Flow: Multiple arrays track cell data (celldata), colors (cellcolors), widths (cellwidth), and merged cells (mergedCellsList object), all synchronized with SCORM for persistence, with modal editing updating individual cell states before final submission.
