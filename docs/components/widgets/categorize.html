Categorize Component

Drag-and-drop interface for categorizing items into different groups.

Overview Architecture Use Cases Component Flow Features

Overview

The Categorize component allows users to organize items into categories through an intuitive drag-and-drop interface.

Interaction Flow:

  • Authoring Phase: Author configures widget settings, uploads content, and saves configuration to backend
  • Student Phase: Student loads widget, interacts with content, and submits answers for validation
  • Validation Phase: System validates student input, provides feedback, and manages attempt tracking
  • Persistence: State is continuously saved to SCORM API for progress tracking and session recovery

Flow Summary:

  • Editor Flow: Authors create 2-4 categories with custom names and colors, add text or image items (0-10 per category), assign correct categories, and select from 6 visual styles.
  • Student Flow: Students drag items from a shuffled pool into appropriate category containers, with visual feedback for drop validation and completion tracking.
  • Validation: Upon submission, items are validated against correct categories with visual indicators (green checkmarks for correct, red borders for incorrect) and attempt-based feedback.
  • Data Flow: Category mappings are stored in the DOM model, synchronized with SCORM for persistence, with Try Again clearing incorrect items or Show Me revealing all correct placements.
