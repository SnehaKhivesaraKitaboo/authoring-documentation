Categorize Component

Drag-and-drop categorization activity where students sort text or image items into labeled categories. Supports 2-4 categories with up to 10 items each and 6 visual styles.

Overview

The Categorize Component presents a drag-and-drop sorting challenge where students classify items into 2-4 labeled categories. Authors configure category titles, drag items (text or image), and correct assignments. Six visual style variants control the visual layout and color scheme. jQuery UI drag-and-drop with touch-punch enables cross-device interaction.

Key Capabilities

  • 2-4 categories with up to 10 items each
  • Text or image drag items configurable per category
  • 6 visual style variants (categorize-style1 through style6)
  • Fisher-Yates shuffle on each load (not saved - reshuffles on reload)
  • Click-to-place alternative for accessibility
  • Individual and generic feedback per item and per attempt
  • Image configuration (display mode, ratio, custom dimensions) per category

Behaviors & Constraints

  • Category limits enforced: addCategory() blocks at 4 categories; deleteCategory() blocks at 2. Both check array length before mutation.
  • Fisher-Yates shuffle: Items are shuffled on every load using Fisher-Yates algorithm. The shuffle is NOT saved — students see different item orders on reload.
  • Image config per category: imageConfiguration (display mode, ratio, custom dimensions) applies to all items in a category, not individual items.
  • jQuery UI + touch-punch: Standard jQuery UI Draggable/Droppable handles desktop; touch-punch adds touch event translation for mobile devices.
  • Click-to-place alternative: Students can click an item then click a category as an alternative to drag-and-drop for accessibility.
  • Try Again behavior: On non-final attempts, correctly placed items lock in place; only incorrectly placed items return to the item bank.
  • SCORM persistence: stateMainatainategorize() serializes placement state for SCORM reporting; state survives page navigation and reload.
