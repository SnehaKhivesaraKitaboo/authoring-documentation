Multiple Choice Question (MCQ) Component

Assessment widget supporting single and multiple-select questions with media support. Features 60 unique visual styles (6 layouts x 10 color presets), Base64-encoded feedback, and comprehensive SCORM tracking.

Overview

The MCQ Component is the primary assessment widget supporting single-select (radio) and multiple-select (checkbox) question formats. It features 60 visual appearances from 6 layout styles multiplied by 10 color presets, plus full custom color override. Individual feedback for each answer option is stored Base64-encoded. Media support includes image, local video, YouTube, Kaltura, and audio.

Key Capabilities

  • Single-select (radio) and multiple-select (checkbox) modes
  • 60 visual styles: 6 layouts x 10 color presets
  • Custom color pickers override preset colors
  • Individual answer feedback Base64-encoded (btoa/atob)
  • Media: image, local MP4, YouTube, Kaltura, Audio.js
  • Sample answer layer via isLayerEnable
  • Shuffle answer options (Fisher-Yates)

Behaviors and Constraints

  • 60 visual styles: The combination of 6 layout styles (card-style1-3 and mcq_full_bleed_style1-3) and 10 color presets (selected_label 1-10) creates 60 unique appearances. Custom colors can further override any preset.
  • Base64-encoded feedback: Individual option feedback is stored as btoa-encoded strings in feedbackinline. Decoded via atob() on display. This prevents casual inspection but provides minimal security.
  • isSingleSelect controls input type: When isSingleSelect=true, radio inputs are rendered; when false, checkbox inputs are rendered. The msOrSsBoolean setting maps to this property.
  • Audio.js media player: Audio files use the Audio.js library for consistent cross-browser playback with custom controls. The player initializes with a 300ms $timeout after the component renders.
  • isLayerEnable sample answer: When enabled, a separate overlay layer shows the correct answer as a visual model before students attempt the question.
