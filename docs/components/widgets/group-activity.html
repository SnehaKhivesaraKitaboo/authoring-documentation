Group Activity (Multipart) Component

Container widget holding up to 35 assessment widgets with centralized submit/reset controls. Supports 3 layouts (vertical scroll, horizontal slider, worked example) with proportional scoring across child widgets.

Overview

The Group Activity Component is a container widget that holds up to 35 other assessment widgets (those with enableGroupInteractivity:true). It provides centralized submit, reset, and try-again controls that operate all child widgets simultaneously. Three layout styles support different pedagogical approaches: vertical scroll, horizontal slider with bullet navigation, and worked example with collapsible sections.

Key Capabilities

  • Holds up to 35 child widgets
  • 3 layout styles: vertical scroll, horizontal slider, worked example
  • Child submit buttons hidden � parent controls all
  • Proportional scoring: MCQ/FIB/MTP partial credit
  • Background color, border radius, outline customization
  • Shuffle questions option (Fisher-Yates)
  • CL client special-cased (window.client == CL)

Behaviors and Constraints

  • Child submit hidden: All submit/reset/try-again buttons in child widgets are hidden via CSS; the parent Group Activity controls provide unified submit, reset, and try-again actions.
  • Submit via CSS class reading: Parent submit clicks each child element matching .submit-btn, then reads CSS classes (.correct, .incorrect) on answer elements to calculate the score.
  • highlightGroupBullets performance: A setInterval running every 1 second continuously checks and updates bullet navigation state for style2. This runs even when no style2 exists on the page.
  • Worked Example forces non-interactive: workedExample styleClass automatically sets isInteractive=no, hiding the submit button and making all child content display-only.
  • CL client special-casing: Multiple behaviors differ when window.client=="CL", including Submit visibility rules in style2 and SCORM reporting format.
  • enableGroupInteractivity required: Only widgets with enableGroupInteractivity:true in their configuration can be dropped into a Group Activity container.
