Component Overview

The Group Activity component enables multi-user collaborative learning through structured multi-part activities with role assignments, individual submissions, and group coordination.

Interaction Flow:

  • Authoring Phase: Author configures widget settings, uploads content, and saves configuration to backend
  • Student Phase: Student loads widget, interacts with content, and submits answers for validation
  • Validation Phase: System validates student input, provides feedback, and manages attempt tracking
  • Persistence: State is continuously saved to SCORM API for progress tracking and session recovery

Flow Summary:

  • Editor Flow: Authors select container styles (1-3), set headers and instructions, drag multiple assessment widgets (MCQ, FIB, True/False, etc.) into the container, configure each child component, and set unified control buttons with max attempts.
  • Student Flow: Students interact with multiple child assessment components within a single container, completing all required responses before unified submission becomes available.
  • Validation: All child components are validated simultaneously upon submission, with individual marking and consolidated feedback, aggregating scores across all children for overall assessment results.
  • Data Flow: Child component states are aggregated to parent Group Activity state, with consolidated SCORM synchronization, Try Again resetting only incorrect children, or Show Me revealing all correct answers across components.
