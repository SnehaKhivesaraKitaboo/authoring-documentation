Overview

The CLIC Component (Sketchpad) embeds an external Carnegie Learning Interactive Content environment within the Kitaboo reader via iframe. Authors provide a CLIC UID that maps to a specific Carnegie Learning activity. Students interact entirely within the embedded iframe; Kitaboo only manages the iframe URL construction and submit state.

Key Capabilities

  • Embeds Carnegie Learning via unique CLIC UID
  • Two-state UI: form input → iframe embed (no revert)
  • Production and staging environment URL routing
  • Auto-submits if clicuid is pre-populated (10ms setTimeout)
  • Introduction text and caption configurable
  • Submit state persisted via issubmitted flag
  • Environment detected via window.clicBaseUrl or platform attribute

Behaviors and Constraints

  • Two-state UI: The component has a form state (UID input) and an iframe state. Once submitted, there is no way to revert to the form state — issubmitted=true is permanent for the session.
  • Auto-submit on load: A 10ms setTimeout fires on initialization; if clicuid is already populated, submitCLIC() is called automatically, bypassing the form state.
  • No input validation: There is no validation on the CLIC UID. Invalid UIDs will load a broken iframe from Carnegie Learning with no error message in Kitaboo.
  • No SCORM from Kitaboo: All scoring and state management is handled by Carnegie Learning. Kitaboo only tracks issubmitted=true.
  • Offline non-functional: Since CLIC requires live Carnegie Learning servers, the component displays nothing in offline or packaged SCORM mode.
  • Environment routing: The iframe URL is determined by window.clicBaseUrl, falling back to window.platform, then article[data-platform] attribute.
