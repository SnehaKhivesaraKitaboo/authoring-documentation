CLIC Component

Click-responsive interactive learning component for engaging content interactions.

Overview Architecture Use Cases Features

Overview

The CLIC (Click-Interaction Learning Component) enables interactive content that responds to user clicks. It supports hotspots, clickable regions, and progressive content revelation to create engaging learning experiences.

User Workflows:

  • Learner: Explores interactive content by clicking hotspots, views associated information or media, explores all regions for discovery-based learning.
  • Author: Creates clickable hotspot regions on images or backgrounds, links content to each hotspot, and configures interaction modes (single, multiple, or ordered sequence).

Component Flow Diagram

This diagram illustrates the complete user interaction flow from authoring to student engagement:

flowchart TD Start([Author Opens Editor]) --> ConfigCLIC[Configure CLIC Component] ConfigCLIC --> SetURL[Set Carnegie Learning URL] SetURL --> ConfigDimensions[Set iframe Dimensions] ConfigDimensions --> SetLabels[Set Question Labels] SetLabels --> SaveCLIC[Save CLIC Configuration] SaveCLIC --> StudentLoad([Student Opens Widget]) StudentLoad --> InitIframe[Initialize External iframe] InitIframe --> LoadExternal[Load Carnegie Learning Content] LoadExternal --> RenderExternal[Render External Interface] RenderExternal --> StudentInteract[Student Interacts with External System] StudentInteract --> ExternalProcess[Carnegie Learning Processes Data] ExternalProcess --> ExternalValidation[External System Validates] ExternalValidation --> PassbackData{Data Available?} PassbackData -->|Yes| ReceiveData[Receive External Results] PassbackData -->|No| MonitorState[Monitor iframe Events] ReceiveData --> UpdateLocal[Update Local State] UpdateLocal --> DisplayStatus[Display Completion Status] DisplayStatus --> SaveState[Save State to SCORM] MonitorState --> StudentInteract SaveState --> DataFlow[Data Flow: External System → KITABOO] DataFlow --> PostMessage[PostMessage Communication] PostMessage --> ParseResponse[Parse External Response] ParseResponse --> MapToSCORM[Map to SCORM Data Model] MapToSCORM --> PersistState[Persist to LMS] PersistState --> Complete([Activity Complete])

Flow Summary:

  • Editor Flow: Authors configure the external Carnegie Learning URL, set iframe dimensions for proper display, and define question labels for the integration.
  • Student Flow: Students interact directly with the embedded Carnegie Learning interface through an iframe, with all processing handled by the external system.
  • Validation: External system performs all validation and scoring, with results passed back to KITABOO via PostMessage API communication protocol.
  • Data Flow: Bidirectional communication between KITABOO and Carnegie Learning systems, mapping external results to SCORM data model for LMS persistence and tracking.
