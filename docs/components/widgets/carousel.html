Carousel Component

Display multiple items in a sliding carousel interface with navigation controls.

Overview UML Class Diagram Sequence Diagram Component Flow Features

Overview

The Carousel component enables navigation through multiple content items with smooth transitions and interactive controls. It supports images, videos, and text content with customizable layouts and auto-play functionality.

Key Capabilities:

  • Multi-slide presentations (2-12 slides)
  • Image, video, and text content support
  • Vertical and horizontal layout options
  • Auto-play with configurable duration
  • Navigation controls and indicators
  • Responsive design for all screen sizes

Interaction Flow:

  • Authoring Phase: Author configures widget settings, uploads content, and saves configuration to backend
  • Student Phase: Student loads widget, interacts with content, and submits answers for validation
  • Validation Phase: System validates student input, provides feedback, and manages attempt tracking
  • Persistence: State is continuously saved to SCORM API for progress tracking and session recovery

Flow Summary:

  • Editor Flow: Authors configure slides (2-12) with titles, text, and media, select navigation style (dots, thumbnails, or progress bar), and set autoplay options with timing controls.
  • Student Flow: Students view slides sequentially or jump to specific slides via navigation controls, with optional automatic advancement and looping functionality.
  • Validation: Content is displayed without assessment validation, focusing on presentation and navigation tracking for completion status.
  • Data Flow: Tracks slide viewing progress and navigation patterns, persisting viewed slide indices and completion status to SCORM for analytics.
