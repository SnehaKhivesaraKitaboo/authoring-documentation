Component Overview

The Sidebar component provides an expandable/collapsible panel for displaying supplementary content, resources, and navigation without disrupting the main content area.

Interaction Flow:

  • Authoring Phase: Author configures widget settings, uploads content, and saves configuration to backend
  • Student Phase: Student loads widget, interacts with content, and submits answers for validation
  • Validation Phase: System validates student input, provides feedback, and manages attempt tracking
  • Persistence: State is continuously saved to SCORM API for progress tracking and session recovery

Flow Summary:

  • Editor Flow: Authors set header and body content text, optionally add inline images, choose background colors, enable outline with custom border colors, and set default state (expanded or collapsed with expandView flag).
  • Student Flow: Students view sidebar in default state, click header to toggle between expanded (down arrow, visible content) and collapsed (right arrow, hidden content) states with CSS transitions.
  • Validation: Non-assessment content display widget with no validation; tracks user interactions (expand/collapse actions) and state persistence for engagement analytics without scoring.
  • Data Flow: Toggle checkbox state is updated on expand/collapse, synchronized with SCORM for state persistence, tracking interaction count as users navigate between expanded and collapsed views throughout the session.
