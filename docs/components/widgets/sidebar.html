Sidebar (Asidebar) Component

Collapsible side panel widget using pure CSS checkbox hack for expand/collapse with zero JavaScript in reader mode. Provides supplementary content that students can show or hide without affecting the main canvas.

Overview

The Sidebar (Asidebar) Component creates a collapsible side panel using a pure CSS checkbox trick — no JavaScript runs in reader mode for expand/collapse. The panel appears alongside the main canvas and contains supplementary content (text, images, links). Authors configure the background color, outline, and content. Students click to expand/collapse without any JS event handlers.

Key Capabilities

  • Pure CSS checkbox hack for collapse/expand — zero JS in reader
  • Collapsed: height:0 opacity:0 visibility:hidden
  • Expanded: height:auto opacity:1 with 0.5s linear transition
  • Custom background color with 5 color presets
  • Unique checkbox ID generated from header+pageNo+savedIndex
  • Mobile media query bug targets p instead of div
  • No SCORM — informational only

Behaviors and Constraints

  • Pure CSS checkbox hack: The expand/collapse is driven by a hidden checkbox input paired with a label element. The checkbox state change triggers CSS adjacent-sibling selectors to animate the content area — no JavaScript runs.
  • Unique checkbox ID: The checkbox input ID is generated by concatenating the header text, page number, and savedIndex to ensure uniqueness across multiple sidebar components on the same page.
  • CSS transition: The collapsed-to-expanded transition animates height from 0 to auto with opacity 0 to 1 over 0.5 seconds using linear easing. The height:auto animation uses a max-height workaround.
  • Icon font codes: The expand/collapse arrows use custom font character codes \108 (right-arrow for collapsed) and \109 (down-arrow for expanded) from a widget icon font.
  • Mobile media query bug: The responsive CSS media query incorrectly targets p elements for layout adjustment instead of div elements, potentially causing display issues with non-paragraph content on mobile.
  • No SCORM: The Asidebar component has no student interaction state — it is a content display widget only. Student expand/collapse preferences are not saved.
