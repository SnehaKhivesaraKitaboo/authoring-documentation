Question Answer (Short/Long Answer) Component

Open-ended response widget supporting three input types: short answer (contenteditable), long answer (textarea), and rich text editor (Cazary basic or Jodit full WYSIWYG) with optional custom keyboard support.

Overview

The Question Answer Component provides three answer input types: Short Answer (contenteditable div, 150 char default), Long Answer (textarea, 5000 char default), and Rich Text Editor (Cazary basic or Jodit advanced WYSIWYG). Content is persisted via data-pretext attribute as Base64-encoded HTML. No automated answer validation � open-ended only; SCORM completion is tracked.

Key Capabilities

  • Short answer: 150 char default via contenteditable
  • Long answer: 5000 char default via textarea
  • Rich Text: Basic (Cazary) or Advanced Jodit WYSIWYG
  • Jodit ELA toolbar: 30+ formatting buttons
  • Content persisted as Base64-encoded data-pretext attribute
  • Custom keyboard for Arabic and non-Latin languages
  • Math via MathLive web component

Behaviors and Constraints

  • No automated validation: The component only tracks completion (SCORM) � there is no mechanism to validate student answers against a model answer. It is a pure open-ended capture tool.
  • Base64 persistence: Student content is saved as Base64-encoded HTML in the data-pretext attribute, allowing HTML formatting to be preserved across page navigations.
  • Jodit ELA toolbar: When formattingValue=ELA, Jodit is configured with 30+ toolbar buttons including bold, italic, lists, tables, and MathLive equation input. Other modes show a reduced toolbar.
  • Cazary iframe editor: The BASIC RTE (Cazary) is an iframe-based editor using document.designMode="on", with simpler formatting options than Jodit.
  • Character count scope: For Jodit/Cazary editors, character counting includes HTML tags in the raw content string, potentially allowing fewer visible characters than the limit suggests.
