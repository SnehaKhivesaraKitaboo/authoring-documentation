Component Overview

The Question Answer component provides text input fields for short-answer and long-answer responses with validation, character limits, and rubric-based scoring.

Flow Summary:

  • Editor Flow: Authors set question text, choose answer type (short, long, or RTE), configure character limits (default 150 for short, 5000 for long), select keyboard type (basic Cazary or advanced Jodit for RTE), and optionally enable custom language keyboards (French, Spanish, German).
  • Student Flow: Students type responses in text fields, textareas, or rich text editors with character counting, special character keyboards when configured, and submit button enabled only when text is present.
  • Validation: No automated scoring; responses are captured and submitted for manual teacher grading, with HTML sanitization for RTE content and character limit enforcement via truncation.
  • Data Flow: Student text responses are stored in fieldData.answerText, encoded as Base64, saved to SCORM suspend_data with response time latency tracking, and marked with dirty flags for unsaved changes until API submission confirms persistence.
