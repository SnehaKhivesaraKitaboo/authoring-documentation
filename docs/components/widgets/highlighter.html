Highlighter Component

Text selection assessment widget where authors pre-mark correct words/phrases and students highlight matching text. Validates selections using window.getSelection() with color-coded correct/incorrect feedback.

OverviewUMLSequenceFlowFeaturesTechnicalData ModelFilesSettingsBehaviors

Overview

The Highlighter Component presents reading passages where students click and drag to highlight text they identify as correct. Authors pre-mark the correct words/phrases in the authoring tool. On submission, student selections are compared to correct spans, with green styling for correct highlights and red for incorrect. The window.getSelection() and Range API handle selection detection.

Key Capabilities

  • Authors pre-mark correct text via highLightStrike() in editor
  • Students select text via click-and-drag (mouseup/touchend)
  • Correct selections shown as .myWords (green) — wrong as .wrongWords (red)
  • 1-5 configurable attempt limit
  • Group activity detection via parent container class check
  • isActivityInprogress flag blocks all interaction during processing

Behaviors and Constraints

  • Author pre-marks correct text: In the authoring UI, authors select text and click highLightStrike() which wraps the selection in span.highLighter elements — these define the answer key.
  • window.getSelection() validation: Student selections are captured via the browser Selection API on mouseup and touchend events, supporting both desktop and mobile.
  • Color feedback: Correct student selections (matching .highLighter spans) get class .myWords (green); incorrect selections (non-matching) get .wrongWords (red).
  • isActivityInprogress lock: During submit processing and animation, isActivityInprogress=true blocks all new text selection events, preventing race conditions.
  • Group activity detection: The component checks for a .group-interactivity-container ancestor to adjust its submit behavior when embedded in a Group Activity.
  • SCORM via saveAction(): State is persisted through the generic saveAction() function which normalizes SCORM data across different SCORM versions.
