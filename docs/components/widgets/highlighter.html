Highlighter Component

Text highlighting tool for marking important content sections.

Overview

The Highlighter component allows users to select and highlight specific text passages with different colors. It's useful for marking important information, identifying key concepts, or categorizing content by color-coding.

Interaction Flow:

  • Authoring Phase: Author configures widget settings, uploads content, and saves configuration to backend
  • Student Phase: Student loads widget, interacts with content, and submits answers for validation
  • Validation Phase: System validates student input, provides feedback, and manages attempt tracking
  • Persistence: State is continuously saved to SCORM API for progress tracking and session recovery

Flow Summary:

  • Editor Flow: Authors add paragraph text, mark sentences to be interactive (1-10), designate correct sentences for highlighting, select highlight color, and configure max attempts with feedback messages.
  • Student Flow: Students click sentences to toggle highlighting on/off, with submit button enabled only when at least one sentence is selected, allowing iterative selection refinement before submission.
  • Validation: Selected sentences are validated against correct answers with visual feedback (green highlights/checkmarks for correct, red highlights/crosses for incorrect, indicators for missed sentences), with scoring calculated.
  • Data Flow: Highlighted sentences array is maintained and synchronized with SCORM, with Try Again clearing only incorrect highlights while preserving correct ones, or Show Me highlighting all correct sentences automatically.
