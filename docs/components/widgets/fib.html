Fill in the Blank (FIB)

Text-based fill-in-the-blank questions with validation.

Overview Architecture Use Cases Features

Overview

The Fill in the Blank (FIB) component provides an interactive learning experience where users complete sentences by filling in missing words or phrases. It supports multiple interaction modes including text input, dropdown selection, and drag-and-drop.

User Workflows:

  • Learner: Reads the sentence, identifies blanks, selects or types answers, submits for validation, and reviews feedback with correct answers.
  • Author: Creates sentences with blanks, defines correct answers and alternatives, configures interaction mode (text/dropdown/drag), and sets validation rules.

Flow Summary:

  • Editor Flow: Authors create sentences (1-10) with blanks, choose input types (dropdown, text input, or drag-drop), configure options (2-6 for dropdowns), set correct answers with optional case sensitivity, and apply visual styles.
  • Student Flow: Students fill blanks by selecting from dropdowns, typing text answers, or dragging items into drop zones, with validation preventing invalid interactions and enabling submission when complete.
  • Validation: Each blank is validated against correct answers with visual feedback (green borders/checkmarks for correct, red borders/crosses for incorrect), calculated scoring, and attempt-based progression.
  • Data Flow: Blank answers are stored in an array and synchronized with SCORM, with Try Again clearing only incorrect blanks or Show Me revealing all correct solutions.
