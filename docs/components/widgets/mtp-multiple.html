MTP Multiple Component

Enhanced Match The Pairs widget supporting one-to-many connections where each Column A item can match up to 3 Column B options. Uses SVG lines and supports configurable per-pair matching type.

OverviewUMLSequenceFlowFeaturesTechnicalData ModelFilesSettingsBehaviors

Overview

MTP Multiple extends the standard Match The Pairs widget to allow each Column A item to connect to 1, 2, or 3 Column B items (configurable per-pair). The key structural difference is that pair2 is an array (not a single object), and subIdentifier distinguishes multiple Column B matches for the same Column A source. displaceFromBottom() aligns Column A items vertically when they have multiple matches.

Key Capabilities

  • Per-pair toggle: single or multiple match mode
  • Each Column A can connect to up to 3 Column B items
  • subIdentifier tracks individual matches within a multiple pair
  • displaceFromBottom() aligns Column A with multiple targets
  • Correct: green 5px left border (#EBF6E7); Incorrect: red 5px left border (#FBE3E3)
  • Max 20 total elements
  • 4 visual style variants (same as MTP)

Behaviors and Constraints

  • pair2 as Array: Unlike standard MTP where pair2 is a single object, MTP Multiple always stores pair2 as an array. Even single-match pairs have a one-element array, allowing the data structure to scale without schema changes.
  • subIdentifier uniqueness: Each pair2 item in the array has a subIdentifier (e.g., B1-1, B1-2) that uniquely identifies it within the pair group for SCORM state tracking.
  • displaceFromBottom() alignment: When a Column A item has multiple Column B matches, displaceFromBottom() calculates the vertical offset to center-align the Column A item with its multiple targets.
  • Feedback color borders: Correct matches get a 5px left border in #EBF6E7 (light green); incorrect matches get #FBE3E3 (light red) — applied as inline CSS after submission.
  • Max 20 total elements: The combined total of all Column A and Column B items across all pairs cannot exceed 20 elements due to SVG rendering performance constraints.
