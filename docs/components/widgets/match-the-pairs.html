Match The Pairs Component

Pair-matching activity with SVG connection lines drawn between Column A and Column B items. Supports drag-drop and click-to-connect interaction, image/text pairs, and 4 visual styles.

OverviewUMLSequenceFlowFeaturesTechnicalData ModelFilesSettingsBehaviors

Overview

The Match The Pairs Component renders two columns (Column A and Column B) of items that students connect by dragging or clicking. SVG path elements draw connection lines between matched pairs. Lines are redrawn on window resize. All-or-nothing scoring: all pairs must be correctly matched for any SCORM credit.

Key Capabilities

  • SVG connection lines drawn as M{x1} {y1} L{x2} {y2} paths
  • Drag-and-drop and click-to-connect interaction modes
  • Text or image items per column (independently configurable)
  • 4 visual style variants (mtpStyle1-4)
  • Lines redrawn on window resize via mtpResize()
  • Shuffle option for Column B items
  • All-or-nothing scoring model

Behaviors and Constraints

  • SVG line drawing: svgDrawLine() creates an SVG path element with format M{x1} {y1} L{x2} {y2} where coordinates are calculated from the center-right of pair1 and center-left of pair2.
  • Window resize redraws: mtpResize() is bound to window.resize and redraws all existing SVG connection lines when the container dimensions change.
  • All-or-nothing scoring: A student must correctly match ALL pairs to receive any SCORM credit. Partial matches score 0.
  • Validation format: Each connection is validated by reading a hidden input formatted as sourceID_targetID and comparing to the correct pair identifiers.
  • Horizontal variant deprecated: The hsMlBoolean="horizontal" variant exists in the data model but is non-functional and visually broken — only "vertical" is supported.
  • Click-to-connect: As an alternative to drag-drop, students can click a Column A item then click a Column B item to create a connection without dragging.
