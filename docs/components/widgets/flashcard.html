Flashcard Component

Flip-card interface for memorization and quick review.

Overview

The Flashcard component provides a digital flip-card interface for memorization and self-paced learning. Users can click or tap to flip between front and back of cards, making it ideal for vocabulary, definitions, and quick review sessions.

Interaction Flow:

  • Authoring Phase: Author configures widget settings, uploads content, and saves configuration to backend
  • Student Phase: Student loads widget, interacts with content, and submits answers for validation
  • Validation Phase: System validates student input, provides feedback, and manages attempt tracking
  • Persistence: State is continuously saved to SCORM API for progress tracking and session recovery

Flow Summary:

  • Editor Flow: Authors create flashcard sets (2-20 cards) with front and back content, optionally add images or videos to either side, select from 2 visual styles, and configure card colors for branding.
  • Student Flow: Students navigate through cards using previous/next controls, click cards to flip between front and back sides, with automatic wrapping from last to first card for continuous study.
  • Validation: Non-assessed learning tool focuses on content exploration, tracking which cards have been viewed and flipped to measure engagement and completion status.
  • Data Flow: Card viewing and flip states are tracked in a viewed cards array, synchronized with SCORM for progress persistence, with progress indicators updated as students explore content.
