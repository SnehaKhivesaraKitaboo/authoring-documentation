Flashcard Component

Flip-card widget with configurable front and back sides supporting text and media. Authors build decks of 2-20 cards with independent front/back content, 2 visual styles, and optional auto-advance.

Overview

The Flashcard Component presents a deck of 2-20 flip cards. Each card has an independently configurable front and back face, each supporting text, images, audio, or video. Students click a card to flip it. Navigation between cards uses bxSlider. Two style variants control the navigation UI (arrows vs. progress bar).

Key Capabilities

  • 2-20 cards per deck
  • Independent front/back content (text or media)
  • Media types: image, audio, video per face
  • 2 visual styles: style-1 (arrows) and style-2 (progress bar)
  • Auto-advance with configurable pause duration (default 3s)
  • Card text limited to 150 characters
  • Theme color and card background color customization

Behaviors and Constraints

  • isBackCardActive context: The settings panel switches context between front and back card configuration via isBackCardActive flag. Controls shown change based on which face is active.
  • Text limit via keydown: The 150-character limit is enforced on keydown events only — paste operations bypass the limit, allowing longer text to be pasted in.
  • Media stops on flip/navigation: When a card is flipped or the slider navigates, active videos are paused via YouTube postMessage and audio elements are paused directly.
  • Global keydown memory leak: A global keydown event listener is registered but never removed when the component is destroyed, accumulating with each component on the page.
  • bxSlider touchEnabled: Touch swiping is disabled (touchEnabled:false) in the editor environment; enabled in reader mode for mobile users.
  • Non-gradable: Flashcards are content presentation tools; no SCORM scoring or student interaction state is tracked.
