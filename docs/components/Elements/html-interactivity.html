HTML Interactivity Element

A ZIP package-based interactive content element that embeds custom HTML/JavaScript interactivities — such as simulations and animations — directly within a page. Supports inline playback, cover image upload, instruction text, header, and caption.

Key Capabilities

  • ZIP file upload (containing index.html + assets)
  • Inline iframe playback on canvas
  • Modal/popup playback mode
  • Cover image upload (displayed before play)
  • Custom width and height settings
  • Optional instruction text
  • Optional header text
  • Optional caption text
  • Outline / background color styling
  • Accessibility alt text
Purpose: The HTML Interactivity element allows authors to embed any custom HTML5-based interactive content (simulations, drag-and-drop activities, animations, games) packaged as a ZIP file. The ZIP must contain an index.html entry point. The element either embeds it inline via <object> or renders a cover image with a launch button.

Playback Modes

Inline Mode

The interactivity is embedded directly on the page canvas using an <object> tag:

<object data="{{fieldData.indexLink}}"
  style="width:100%;height:100%;">
</object>

Authors can interact with the content directly on the authoring canvas.

Cover Image Mode

A static cover image is shown on the canvas. In the reader, clicking the image launches the interactivity in a modal or full-screen view. The cover image is uploaded separately from the ZIP.

ZIP Upload Validation

When a ZIP file is uploaded, the system validates it contains a valid index.html entry point. If validation fails, a modal error dialog is shown:

Modal ID: #html-interactivity-valid
Error Message: "Please try a valid zip file."
