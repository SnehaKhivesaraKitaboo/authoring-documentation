Header Element

A flexible section-heading component supporting H2–H6 semantic levels and subtitle text, with 4 visual styles and optional decorative icons.

Key Capabilities

  • Semantic heading levels: H2, H3, H4, H5, H6, and Subtitle
  • 4 visual style cards (card-style-1 to card-style-4)
  • 7 optional decorative icon shapes
  • Custom border/accent color via componentColor
  • Inline contenteditable text
  • Apply All / Ignore Apply All toggle
  • TOC integration
Purpose: The Header element provides semantic section headings (H2–H6) inside page content. It is distinct from the Title element (which is the full chapter heading) — Headers are used for in-page section breaks, sub-topics, and content organization within a page.

Template Switching Pattern

The Header element uses ng-if to render the correct semantic HTML element based on fieldData.settings.headerClassData. All heading variants share the same ng-model binding (fieldData.headerDataText), so switching heading level preserves the text.

header.html — Semantic element switching
<h2 ng-if="fieldData.settings.headerClassData == 'h2-style'"
    ng-model="fieldData.headerDataText" contenteditable="true"
    class="header-data tocTextContent" ...>
</h2>
<h3 ng-if="fieldData.settings.headerClassData == 'h3-style'"
    ng-model="fieldData.headerDataText" contenteditable="true" ...>
</h3>
<!-- ...h4, h5, h6, p (subtitle) -->
