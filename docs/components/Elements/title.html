Title Element

A full-page chapter heading component that supports background images, color overlays, author name, chapter number labels, and multiple visual style variants.

  • Multiple visual style variants (card-style-1..4, full-bleed-1/2)
  • Background image with crop/mask support

Purpose: The Title element is the primary chapter-level heading component. It occupies the full width of the page canvas and supports rich visual customization including background images (with built-in Cropper.js), color themes, and multiple layout styles. It is the first element authors typically place when creating a new chapter page.

How It Works

When an author drags the Title tile from the Elements panel:

  1. The ngcontroller.js reads the widget definition from config.js, fetches default/header.json as the initial fieldData, and loads chapter-heading.html.
  2. The HTML is $compiled against a new child scope carrying the JSON as fieldData, and injected into the page canvas.
  3. The chapterHeadingNewTemplate directive's link function fires: undo stacks are initialized, click handlers are bound, the first-load Apply All check runs, and the Cropper.js image widget is initialized if needed.
  4. The author clicks the element to select it — the right-side settings panel loads chapter-heading-settings-panel.html, wired to con.currSettings.
  5. Every change to fieldData is auto-persisted via savedJson and can be saved via the Save button.

Initialization (link function)

titleNew-directive.js — Key Behaviors
// 1. Undo stack setup
scope.undoHeaderTitleTextStack = [];
scope.undoTitleAuthorNameStack = [];

// 2. First-load ApplyAll check
if (element.parents('.template-main-body').eq(0).hasClass('first-time-load')) {
  if (scope.$parent.ApplyToAllStyleSettingJson['chapter-headingNew-h1'].styleSettingFlag) {
    // Apply stored style settings, preserve text content
    var scopeCopy = angular.copy(scope.$parent.ApplyToAllStyleSettingJson[...].styleSettingJson);
    scope.fieldData = scopeCopy;
  }
}

// 3. Font size normalization
setTimeout(() => {
  element.find('.headerTitle').css('font-size', fontValue + 'rem');
}, 10);

// 4. Click → load settings panel
$(element).parents('.sd-item').bind('click', function(e) {
  con.currSettings = con.savedJson[pageNo][uniqueId].settings;
  con.settingsUrl = templateScope.settingsPath;
  con.displaySettingPanel();
});

Image Upload & Crop

The Title element integrates Cropper.js for background image manipulation:

  • Author clicks "Upload" in settings → file input triggers
  • Image is uploaded to server via ShellService.httpServiceCall(POST)
  • Cropper initialized with aspectRatio: 2/1, drag mode
  • On "Crop" click: canvas exported as JPEG → re-uploaded → URL stored in fieldData.settings.imageUrl
  • Original image preserved in data-og-img attribute

Known Behaviors & Notes

Apply All Behavior: When Apply All is active and a new Title element is dropped, it inherits all style settings from the stored Apply All JSON, but the text content fields (headerTitleText, authorName, headerChapterNo, chapterheadingTitleText) are always reset to defaults to avoid copying content.
TOC Integration: The Title element's headerTitleText is registered in the Table of Contents. The directive calls con.updateActivePageToc() on click and highlights the matching TOC item.
Duplicate Component: When a Title is duplicated (data-duplicatecomp="true"), the text content is preserved from the original even when Apply All is active.
